# Постоянное хранилище для Vercel

На Vercel папка проекта доступна только для чтения. Поэтому запись в `src/data/*.json` после деплоя невозможна. Проект теперь использует два режима:

- локально без Redis — чтение и запись в JSON-файлы;
- на Vercel с Redis — постоянное хранение пользователей, ролей, owner-пароля, правил, changelog, статей и просмотров.

## Подключение Upstash Redis

1. Открой проект в Vercel.
2. Перейди в **Storage** или **Integrations / Marketplace**.
3. Выбери **Upstash Redis** и подключи базу к проекту.
4. Разреши интеграции добавить переменные окружения для **Production**. При необходимости также включи **Preview**.
5. Убедись, что появились одна из пар переменных:
   - `UPSTASH_REDIS_REST_URL` и `UPSTASH_REDIS_REST_TOKEN`, либо
   - `KV_REST_API_URL` и `KV_REST_API_TOKEN`.
6. В разделе **Settings → Environment Variables** проверь, что `AUTH_SECRET` задан длинной случайной строкой.
7. Выполни новый деплой после добавления переменных.

## Первый запуск

При первом обращении к каждому разделу Redis автоматически заполнится текущими данными из файлов:

- `src/data/users-db.json`
- `src/data/roles-db.json`
- `src/data/owner-db.json`
- `src/data/rules-db.json`
- `src/data/changelog-db.json`
- `src/data/articles-db.json`
- `src/data/post-views.json`

После этого все изменения из админ-панели сохраняются в Redis и не пропадают при следующих деплоях.

## Локальная разработка с Redis

Можно скопировать `.env.example` в `.env.local` и заполнить данные Upstash. Если Redis-переменные не заданы, локально проект продолжит работать с JSON-файлами.

## Важно

- Не добавляй `.env.local` в GitHub.
- Не меняй `DATA_STORE_PREFIX` после первого запуска, иначе приложение увидит другое пространство ключей как новую базу.
- Preview и Production могут использовать одну базу. Для полной изоляции назначь им разные значения `DATA_STORE_PREFIX` в настройках Vercel.
